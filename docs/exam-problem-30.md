# 전기기능장 PLC 79회 2과제 모의고사 30 분석

## 문제 개요
- 출처: 뱅가의전기놀이터, 모의고사 30
- 파일: `전기기능장예제/전기기능장_PLC_2과제_모의고사_30_답.pdf`(3p), `_답_IL.pdf`(8p)
- 난이도: ★★★★★ (최대공약수/최소공배수 + FIWR리스트 + SORT + SCH 검색)
- 총 스텝: 220개 (21렁)
- 특징: **타이머 미사용!** 스캔 카운터(ADD)로 약수/배수를 순차 생성

## 문제 설명
- PBA(P) → N 저장, PBB(P) → M 저장
- PBC(P) → K 토글 (1~2 입력제한, FF 플립플롭)
- K=1: 최소공배수(LCM), K=2: 최대공약수(GCD)
- SSC(P) → 동작 시작 (조건: N>0 AND M>0 AND PBC동작)
- SSC(N) → 리셋/정지
- 결과값 < 32: 2진수로 램프 지속점등
- 결과값 >= 32: 동작하지 않음 (소등)

### 램프-비트 대응
| 램프 | 비트 | 가중치 |
|------|------|--------|
| PLA | D00000.4 | 2^4 = 16 |

| PLC | D00000.2 | 2^2 = 4 |
| PLD | D00000.1 | 2^1 = 2 |

## 핵심 알고리즘 ★★★★★

### 1단계: K 모드 선택 (FF 플립플롭)
```
ANDP PBC → FF D00003.0          ; PBC 누를 때마다 0↔1 토글
AND NOT D00003.0 → OUT 최대공약수  ; D00003.0=0 → GCD 모드 (K=2)
AND D00003.0 → OUT 최소공배수      ; D00003.0=1 → LCM 모드 (K=1)
```
★ K=1(홀수 누름)→최소공배수, K=2(짝수 누름)→최대공약수

### 2단계: 스캔 카운터로 약수/배수 생성
```
ADD D00010 1 D00010    ; ★ 타이머 대신 매 스캔마다 1씩 증가!

; GCD 모드: 나누기로 약수 판별
DIV N D00010 D00015    ; N ÷ 카운터 → 몫=D00015, 나머지=D00016
DIV M D00010 D00017    ; M ÷ 카운터 → 몫=D00017, 나머지=D00018

; LCM 모드: 곱하기로 배수 생성
MUL N D00010 D00015    ; N × 카운터 → D00015
MUL M D00010 D00017    ; M × 카운터 → D00017
```

### 3단계: FIWR로 리스트 구축
```
; GCD: 나머지=0인 카운터값(약수) 저장
AND= D00016 0 → FIWR D00010 D00100   ; N의 약수 → D00100 큐
AND= D00018 0 → FIWR D00010 D00120   ; M의 약수 → D00120 큐

; LCM: 배수값 저장
FIWR D00015 D00100    ; N×카운터 → D00100 큐
FIWR D00017 D00120    ; M×카운터 → D00120 큐
```

**예시 (N=12, M=8, GCD 모드):**
- 카운터 1~12: N의 약수 = {1,2,3,4,6,12}, M의 약수 = {1,2,4,8}
- D00100 큐: [6개] 1,2,3,4,6,12
- D00120 큐: [4개] 1,2,4,8

### 4단계: 정지 조건 (OR LOAD/AND LOAD 패턴)
```
; GCD: 카운터가 N/M에 도달하면 정지 (약수는 자기 자신까지)
(D00010=N AND 최대공약수) OR (D00010=M AND 최소공배수) → SET 일번정지
(D00010=M AND 최대공약수) OR (D00010=N AND 최소공배수) → SET 이번정지
일번정지 AND 이번정지 → SET 입력정지
```
★ IL에서 OR LOAD/AND LOAD 블록결합 패턴 사용!

### 5단계: GCD용 SORT (내림차순 정렬)
```
MOVP D00100 D00200     ; 약수 개수
MOVP 1 D00201          ; 방향=1 (내림차순)
MOVP 1 D00202          ; 단위=1 워드
SORT D00101 D00200 D00202 D00203 D00204
```
- 큰 약수부터 검색하기 위해 내림차순 정렬
- D00100=1 (약수가 1뿐=소수): SORT 불필요 → 최대공약수시작2로 바이패스
- ★ LCM 모드에서는 SORT 생략 (배수가 이미 오름차순)

### 6단계: SCH 검색으로 공통값 찾기
```
MOVP 1 Z001                                    ; 검색 시작 인덱스
SCH D00100[Z001] D00121 D00150 D00120          ; ★ 핵심!
; S1 = D00100[Z001]: N의 약수/배수 (인덱스로 순회)
; S2 = D00121: M의 약수/배수 데이터 시작
; D1 = D00150: 결과(첫 위치)
; D2 = D00120: M 리스트 포인터(개수)

AND< Z001 D00100 → ADD Z001 1 Z001            ; 다음 항목으로 이동

AND= D00151 1 → (찾음!)
  AND< D00000 32 → MOVP D00150 Z002
                    MOVP D00120[Z002] D00000    ; ★ 결과값 → 출력
  AND>= D00000 32 → MOVP 0 D00000             ; ≥32 → 소등
SET 찾기중지
```

**GCD 검색 과정 (N=12, M=8):**
- N약수 내림차순: {12,6,4,3,2,1}
- Z001=1: SCH 12 in {1,2,4,8} → D00151=0 (없음) → 다음
- Z001=2: SCH 6 in {1,2,4,8} → D00151=0 → 다음
- Z001=3: SCH 4 in {1,2,4,8} → D00151=1 (찾음!) → GCD=4 ✓

**LCM 검색 과정 (N=7, M=4):**
- N배수: {7,14,21,28}
- Z001=1: SCH 7 in {4,8,12,16,20,24,28} → 없음 → 다음
- Z001=2: SCH 14 in ... → 없음
- Z001=3: SCH 21 → 없음
- Z001=4: SCH 28 → 찾음! → LCM=28 ✓

### 7단계: 출력 (이중반전 패턴)
```
LOAD NOT D00000.4 / NOT / OUT P00040  (PLA)
LOAD NOT D00000.3 / NOT / OUT P00041  (PLB)
LOAD NOT D00000.2 / NOT / OUT P00042  (PLC)
LOAD NOT D00000.1 / NOT / OUT P00043  (PLD)
LOAD NOT D00000.0 / NOT / OUT P00044  (PLE)
```

## 새로 배운 명령어/패턴

### 1. FF (Flip-Flop)
```
FF D00003.0    ; 실행할 때마다 0→1→0→1 토글
```
- PBC를 누를 때마다 K=1↔K=2 전환
- SET/RST 없이 한 명령어로 토글 구현

### 2. ANDN (AND N-edge, 음변환)
```
ANDN SSC    ; SSC를 놓을 때(1→0) 리셋
```

### 3. ADD 스캔 카운터 (타이머 대체)
```
ADD D00010 1 D00010    ; 매 스캔마다 +1
```
- 타이머(TON) 대신 스캔 주기를 이용한 순차 카운터
- PLC 스캔이 매우 빠르므로 거의 즉시 전체 약수/배수 생성

### 4. DIV 나머지 활용 (약수 판별)
```
DIV N D00010 D00015    ; D00015=몫, D00016=나머지
AND= D00016 0          ; 나머지=0 → 약수!
```
- DIV의 부산물(D+1=나머지)을 활용한 약수 판별

### 5. SORT + D00203.0 완료 신호
```
SORT D00101 D00200 D00202 D00203 D00204
; D00203.0 = SORT 완료 시 1로 설정됨 (자동)
```
- SORT의 5번째 파라미터(D00204)까지 포함
- SORT 완료 신호를 AND 조건으로 활용

### 6. SCH 인덱스 어드레싱 검색
```
SCH D00100[Z001] D00121 D00150 D00120
```
- 검색 대상을 인덱스(Z001)로 순회하면서 검색
- D00151=1이면 일치 발견, D00150=첫 위치

## 이 문제의 핵심 통찰
1. **타이머 0개**: 스캔 카운터(ADD)로 모든 연산 수행 → 결과가 거의 즉시 출력
2. **FIWR 동적 리스트**: 약수/배수를 실시간으로 배열에 추가
3. **SORT→SCH 파이프라인**: 정렬 후 검색으로 GCD 찾기 (유클리드 호제법 대신 brute force)
4. **FF 토글**: PBC 1회 누름/2회 누름으로 모드 선택 (입력제한 1~2)
5. **DIV 나머지 활용**: 나눗셈의 부산물(D+1)로 약수 판별
