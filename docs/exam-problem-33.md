# 전기기능장 PLC 79회 2과제 모의고사 33 분석

## 문제 개요
- 출처: 뱅가의전기놀이터, 모의고사 33
- 파일: `전기기능장예제/전기기능장_PLC_2과제_모의고사_33_답.pdf`(3p), `_답_IL.pdf`
- 난이도: ★★★★★ (실습과제 11-20 대비 매우 높음)

## 문제 설명
- SSA,SSB로 시스템 시작
- PBA로 N값 입력 (1~5), PBB로 M값 입력 (1~5)
- PBC 누르면 실행 (재동작 가능)
- 5개 램프 PLA~PLE (= 위치 1~5)
- N과 M 사이 구간의 램프는 **동시 점등** 후 **1초씩 지연 소등**
- N > M: PLA→PLE 방향 동작 (모드A, 역방향)
- M > N: PLE→PLA 방향 동작 (모드B, 정방향)
- SSC로 정지 및 리셋
- 1칸 = 1초

## 프로그램 구조 (답안 래더 3페이지)

### Rung 1: 시스템 시작
```
LOAD NOT SSA(P00003) AND SSB(P00004) → OUT D06241.0(과제2)
```

### Rung 5: SSC 리셋
```
과제2(D06241.0) AND SSC(P00005, P-edge) →
  FMOVP 0 D00000 10000    ; D00000~D09999 전부 0으로 클리어
  FMOVP 0 Z000 10          ; Z000~Z009 클리어
  BRSTP M00000 1000         ; M00000~M00999 전부 리셋
```

### Rung 20: 리셋 플래그 처리
```
리셋(D06241.5) →
  FMOVP 0 D00010 10000
  FMOVP 0 Z000 10
  BRSTP M00000 1000
```

### Rung 33: 입력부
```
과제2(D06241.0) AND NOT 동작중입력금지(D06241.1):
  ├─ PBA(P00000, P-edge) AND (D00001 < 5) → INCP D00001    ; N 증가
  ├─ PBB(P00001, P-edge) AND (D00001 < 5) → INCP D00002    ; M 증가 (★ N<5 조건!)
  └─ PBC(P00002, P-edge) AND ANDG> N 0 2 AND AND<> N M →
       ; ANDG>: N>0 AND M>0 동시 확인, AND<>: N≠M 확인
         SET D06241.1(동작중입력금지)
         SET D06241.2(동작)
```

### Rung 62: 모드 판정
```
과제2(D06241.0) →
  MAX D00001 D00005 2       ; MAX(N,M) → 큰수(D00005)
  MIN D00001 D00006 2       ; MIN(N,M) → 작은수(D00006)
  (D00001 > D00002) → SET D06241.3(모드A, 역방향)   ; N > M
  (D00002 > D00001) → SET D06241.4(모드B, 정방향)   ; M > N
```

### Rung 82: 타이머 + 포인터 이동 (JMP/LBL 구조)
```
동작(D06241.2) AND NOT T0001 → TON T0001 10    ; 1초 자동리셋 타이머

; 초기화 (MOVP = P-edge, 모드 플래그 ON 순간 1회만 실행)
모드A(D06241.3) → MOVP 5 Z001, MOVP 6 Z000    ; 역방향: Z001=5(PLE), Z000=6
모드B(D06241.4) → MOVP 1 Z001                   ; 정방향: Z001=1(PLA), Z000=0(리셋값)

; 매 1초마다 포인터 이동
T0001 AND 모드B → INCP Z001, INCP Z000          ; 정방향: 포인터 증가
T0001 AND 모드A → DECP Z001, DECP Z000          ; 역방향: 포인터 감소
```
※ 래더에서는 라벨로 표시되지만, IL에서는 MPUSH/MLOAD/MPOP 스택 분기로 변환됨

### Rung 114: 램프 이동 제어
```
동작(D06241.2) →
  RST M00000[Z000]           ; 뒤쪽 포인터 위치 램프 소등 (인덱스 어드레싱)
  SET M00000[Z001]           ; 앞쪽 포인터 위치 램프 점등

  (Z001 = 6) AND 모드B → OR LOAD →
  (Z001 = 0) AND 모드A → OR LOAD → OUT 리셋(D06241.5)  ; 양쪽 모두 리셋 트리거
```

### Rung 131: 중간 램프 동시점등 (5개 블록 반복)
위치 1~5 각각에 대해:
```
동작(D06241.2) AND <=3(작은수, [위치], 큰수)    ; 작은수 <= 위치 <= 큰수 범위 확인
  ├─ (Z001 = 작은수) AND 모드B → SET M0001x     ; 정방향: 앞 포인터가 MIN 도달 시
  └─ (Z001 = 큰수) AND 모드A → SET M0001x       ; 역방향: 앞 포인터가 MAX 도달 시
```
→ 범위 내 모든 램프가 **동시에** SET됨

### Rung 204: 중간 램프 순차 소등
```
동작(D06241.2) → RST M00010[Z000]    ; 뒤쪽 포인터가 지나가며 중간 플래그도 소등
```

### Rung 208~224: 최종 출력 (OR 합성)
```
; 래더: M00001 OR M00011 → P00040 (PLA)
; IL (De Morgan OR): LOAD NOT M00001 / AND NOT M00011 / NOT / OUT P00040
M00001 OR M00011 → P00040 (PLA)
M00002 OR M00012 → P00041 (PLB)
M00003 OR M00013 → P00042 (PLC)
M00004 OR M00014 → P00043 (PLD)
M00005 OR M00015 → P00044 (PLE)
```
이동 플래그 OR 중간유지 플래그 = 최종 출력

### Rung 228: END

## 핵심 알고리즘: 이중 포인터 스위프

```
  Z001 (앞쪽, 점등)          Z000 (뒤쪽, 소등)
      ↓                          ↓
  ○ ○ ● ● ● ○ ○          ○ ○ ○ ● ● ○ ○
      → 전진                      → 전진 (1초 뒤따름)
```

### Mode B (M>N, 정방향) 실행 예시: N=2, M=4

| 시간 | Z001 | Z000 | PLA | PLB | PLC | PLD | PLE | 이벤트 |
|------|------|------|-----|-----|-----|-----|-----|--------|
| 0s | 1 | 0 | ON | - | - | - | - | 스위프 시작 |
| 1s | 2 | 1 | OFF | **ON** | **ON** | **ON** | - | Z001=MIN→중간 동시점등 |
| 2s | 3 | 2 | - | OFF | ON | ON | - | PLB 소등 |
| 3s | 4 | 3 | - | - | OFF | ON | - | PLC 소등 |
| 4s | 5 | 4 | - | - | - | OFF | ON | PLD 소등, PLE 스위프 |
| 5s | 6 | 5 | - | - | - | - | OFF | 완료→리셋 |

### Mode A (N>M, 역방향) 실행 예시: N=4, M=2

| 시간 | Z001 | Z000 | PLA | PLB | PLC | PLD | PLE | 이벤트 |
|------|------|------|-----|-----|-----|-----|-----|--------|
| 0s | 5 | 6 | - | - | - | - | ON | 스위프 시작 |
| 1s | 4 | 5 | - | **ON** | **ON** | **ON** | OFF | Z001=MAX→중간 동시점등 |
| 2s | 3 | 4 | - | ON | ON | OFF | - | PLD 소등 |
| 3s | 2 | 3 | - | ON | OFF | - | - | PLC 소등 |
| 4s | 1 | 2 | ON | OFF | - | - | - | PLB 소등, PLA 스위프 |
| 5s | 0 | 1 | OFF | - | - | - | - | 완료 |

## 새로 배운 명령어/개념

### 1. D 레지스터 비트 어드레싱
```
D06241.0  → D06241의 비트0
D06241.1  → D06241의 비트1
...
D06241.5  → D06241의 비트5
```
M 릴레이처럼 개별 비트를 플래그로 사용 가능

### 2. 인덱스 레지스터 (Z000~Z009)
```
M00000[Z000]  → M(00000 + Z000의 현재값)
M00010[Z001]  → M(00010 + Z001의 현재값)
```
배열처럼 동적 주소 접근. Z 값이 3이면 M00003 접근.

### 3. FMOVP (Fill Move, P-edge)
```
FMOVP 0 D00000 10000    ; D00000~D09999를 0으로 일괄 초기화
FMOVP 0 Z000 10          ; Z000~Z009를 0으로 초기화
```
- 대량 초기화에 사용
- P 접미사: 양변환(rising edge)에서만 1회 실행

### 4. MOVP (Move, P-edge)
```
MOVP 5 Z001    ; Z001에 5를 저장 (1회)
MOVP 1 Z001    ; Z001에 1을 저장 (1회)
```

### 5. INCP / DECP (Increment/Decrement, P-edge)
```
INCP D00001    ; D00001 값 1 증가
DECP Z001      ; Z001 값 1 감소
```

### 6. MAX / MIN
```
MAX D00001 D00005 2    ; D00001과 D00002 중 큰 값 → D00005
MIN D00001 D00006 2    ; D00001과 D00002 중 작은 값 → D00006
```
3번째 파라미터 = 비교 대상 개수

### 7. BRSTP (Batch Reset, P-edge)
```
BRSTP M00000 1000    ; M00000~M00999 일괄 리셋
```

### 8. <=3 (3항 범위 비교)
```
<=3 D00006 3 D00005    ; D00006(작은수) <= 3 <= D00005(큰수) → ON/OFF
```
범위 내 포함 여부 확인

### 9. JMP / LBL (점프 / 라벨) - 래더 전용
```
JMP 1     ; 라벨 1로 점프 (중간 렁 건너뜀)
LBL 1     ; 라벨 1 (점프 도착지)
```
⚠️ 래더에서는 라벨로 표시되지만, IL 변환 시 MPUSH/MLOAD/MPOP 스택 분기로 대체됨

### 10. AND<> (부등 비교)
```
AND<> N M    ; N ≠ M → ON
```

## 프로그램 아키텍처 패턴

실습과제(11-20)와의 차이:

| 항목 | 실습과제 11-20 | 기능장 모의고사 33 |
|------|---------------|-------------------|
| 제어 방식 | 타이머+비교 | 인덱스+포인터 |
| 데이터 타입 | 비트(M,P,T) | 워드(D)+비트+인덱스(Z) |
| 입력 | 고정 (PB1,PB2) | 가변 (N,M 카운팅) |

| 프로그램 크기 | ~20줄 IL | ~230줄 래더 (228 rung) |

## 핵심 설계 패턴 정리

1. **D 비트 플래그 관리**: 시스템 상태를 D 레지스터 비트로 관리 (D06241.0~.5)
2. **이중 포인터 스위프**: Z001(앞)으로 점등, Z000(뒤)으로 소등 → 이동 효과
3. **2레이어 출력**: 이동 플래그(M0000x) + 중간 유지 플래그(M0001x) → OR → 출력
4. **인덱스 어드레싱**: M00000[Z000]으로 동적 릴레이 접근
5. **래더: JMP/LBL, IL: MPUSH/MLOAD/MPOP**: 모드별 실행 코드 분기
6. **FMOVP/BRSTP 일괄 초기화**: 대량 디바이스 한 번에 클리어
